Module 2 Study Guide · Technology, Prototyping & Execution · Sep 7 – Sep 25

Deliverable: Working Prototype of your top-priority use case, built with low-code/no-code tools. Due ~Sep 25.

Topic 1 — GenAI & agentic fluency at executive depth (Week 5)

You need to reason about architecture without writing code: which pattern fits the use case, where humans stay in the loop, what can fail and how you’d know.

Study:

  • Anthropic, Building Effective AI Agents — the five workflow patterns + when to use a true agent vs. a workflow. Core reading; re-read even if done in prep. https://www.anthropic.com/research/building-effective-agents
  • Anthropic, Writing Effective Tools for AI Agents — what makes a tool usable by an agent; shapes how you spec integrations. https://www.anthropic.com/engineering/writing-tools-for-agents
  • Vellum, The 2026 Guide to AI Agent Workflows — emerging architectures and design patterns, tool-agnostic. https://www.vellum.ai/blog/agentic-workflows-emerging-architectures-and-design-patterns
  • Automation Anywhere, What are Agentic Workflows? — enterprise operating-model angle the course emphasizes. https://www.automationanywhere.com/rpa/agentic-workflows

Concepts to master: orchestration layer (execution order, output passing, failure handling, parallel sync), MCP as the integration standard, human-in-the-loop checkpoints, evaluator/guardrail loops, traceability.

Exercise: sketch your use case as boxes-and-arrows — agents, tools, data sources, human checkpoints, failure paths. This sketch IS the architecture section of the prototype writeup.

Topic 2 — Low-code tool selection (Week 5–6)

Pick based on where the demo must live and what it must touch, not feature counts.

Study:

  • Budibase, 11 Low & No-Code AI Agent Builders for 2026 — broad survey. https://budibase.com/blog/ai-agents/no-code-ai-agent-builders/
  • Bravent, n8n vs Copilot Studio — the key tradeoff: workflow-controlled (you decide when the LLM is called) vs. agent-controlled (model decides). https://www.bravent.net/en/news/n8n-vs-copilot-studio-which-one-should-i-choose-to-build-generative-ai-agents/
  • Botpress, 7 Best Low-Code AI Agent Platforms in 2026. https://botpress.com/blog/low-code-ai-agent-platforms

Decision shortcuts:

  • Org lives in Microsoft 365 → Copilot Studio (tenant controls, native connectors, child agents, MCP support)
  • Integration-heavy pipeline, self-host/security-first → n8n (1300+ connectors, source-available; now integrates with Microsoft Agent 365 for governance)
  • Fast conversational demo → Claude with MCP connectors / Cowork

Exercise: 30-minute spike in 2 tools with a trivial version of your use case; pick by which spike felt closer to done.

Topic 3 — Building the prototype (Week 6–7)

A prototype that persuades stakeholders shows: real(istic) data in → visible agent reasoning/steps → useful output → a governance hook (audit log, approval gate, or refusal behavior). The governance hook is what separates an executive prototype from a demo toy — and it sets up Module 3.

Scope discipline: one workflow, one happy path, one failure path handled gracefully. Resist adding use cases.

Demo prep checklist:

  • 90-second narrative: pain → agent does X → outcome → what scaling requires
  • Show the failure path on purpose (builds trust)
  • Name the governance hooks aloud
  • Have the architecture sketch as a backup slide

Consultant 1-on-1 (book by Week 6)

Bring: architecture sketch, tool choice + why, the ugliest problem you hit. Ask: “what would make this credible to a skeptical CIO?”
